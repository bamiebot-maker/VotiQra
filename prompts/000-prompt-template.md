# VotiQra AI Implementation Prompt Template

## Objective

Complete the requested feature according to the VotiQra engineering standards.

This project is a production-grade mobile application built with React Native and Expo.

You are expected to behave as a senior software engineer.

---

# Required Reading

Before writing a single line of code, read these documents in order:

1. docs/PROJECT_PRINCIPLES.md
2. docs/AGENTS.md
3. docs/ARCHITECTURE.md
4. docs/PRODUCT.md
5. docs/USER_EXPERIENCE.md
6. docs/NAVIGATION.md
7. docs/DESIGN_SYSTEM.md

These documents are the source of truth.

Do not violate them.

---

# Before Coding

Before generating code:

- Explain your implementation plan.
- Identify the files that will be created.
- Identify the files that will be modified.
- Mention any dependencies required.
- Explain why those dependencies are necessary.

Do not start coding until the plan is complete.

---

# Coding Standards

Follow all project conventions.

Always:

- Use strict TypeScript.
- Use Expo Router.
- Use NativeWind.
- Use Zustand only for global client state.
- Use TanStack Query for server state.
- Use React Hook Form with Zod for forms.
- Keep components reusable.
- Separate UI from business logic.
- Follow the design system.
- Keep code modular.

---

# Constraints

Do NOT:

- Rewrite unrelated files.
- Change architecture.
- Hardcode colors.
- Hardcode spacing.
- Introduce unnecessary packages.
- Use any unless absolutely necessary.
- Disable lint rules.
- Disable TypeScript rules.
